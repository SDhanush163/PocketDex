# ADR-001: Project Initialization

![Status](https://img.shields.io/badge/Accepted-success)
![Date](https://img.shields.io/badge/13%2F07%2F2026-blue)

## Context

Before development begins, the project's initial technologies and conventions need to be established.

## Decision

The project will use:

- Vite
- React
- TypeScript
- npm
- Git and GitHub

Project documentation will be maintained within the repository.

## Rationale

The selected technologies provide a stable and widely adopted foundation for the application. Keeping documentation within the repository ensures that design decisions and implementation remain version controlled together.

## Consequences

### Benefits

- Consistent project foundation.
- Documentation evolves alongside the codebase.

### Trade-offs

- Documentation requires ongoing maintenance.
